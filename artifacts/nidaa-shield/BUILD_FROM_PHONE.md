# دليل بناء "نداء شايلد" من الهاتف

> هذا الإصدار يحتوي على وحدة Android أصلية (`VpnService`) — لذلك يجب البناء عبر EAS Build وليس Expo Go. التطبيق سيطلب إذن VPN ويفعّل الحماية ذاتياً بدون أي خروج للمستخدم من التطبيق.

## الخطوات

1. **افتح GitHub من المتصفح**
   - تأكد أن آخر تحديث وصل إلى المستودع `15coder/Nidaa-Shield`.

2. **افتح [expo.dev](https://expo.dev) وسجّل الدخول**

3. **اذهب إلى المشروع `nidaa-shield`** ← تبويب **Builds**

4. **اضغط "Build from GitHub"** واملأ:
   - **Repository**: `15coder/Nidaa-Shield`
   - **Branch**: `main`
   - **Base directory**: `artifacts/nidaa-shield` *(مهم جداً)*
   - **Platform**: Android
   - **Build profile**: `preview`

5. **اضغط Confirm وانتظر**
   - سيقوم EAS بـ:
     - تثبيت الحزم
     - تشغيل `prebuild` لتوليد مشروع Android (سيكتشف وحدة `nidaa-vpn` تلقائياً)
     - بناء APK كامل مع `VpnService`

6. **عند انتهاء البناء**:
   - ستحصل على رابط APK
   - حمّله على هاتفك وثبّته

## الاستخدام بعد التثبيت

1. افتح التطبيق.
2. اضغط على أي وضع حماية.
3. سيظهر **مربع إذن VPN من Android** — اضغط "موافق".
4. الحماية تبدأ فوراً، ويظهر إشعار دائم في شريط الإشعارات.
5. لإيقاف الحماية: اضغط زر "إيقاف" داخل التطبيق.

## كيف يعمل التطبيق تقنياً

- يستخدم خدمة `VpnService` الأصلية في Android لإنشاء واجهة شبكة افتراضية.
- يضع خادم DNS المختار للوضع كـ DNS overrride على مستوى النظام بأكمله.
- بدون اعتراض حقيقي للترافيك (Pass-through) — الإنترنت يبقى سريعاً وطبيعياً.
- كل التطبيقات على الهاتف ترسل استعلامات DNS عبر الخادم المحدد، فيتم حظر الإعلانات/المحتوى الضار/التتبع تلقائياً.
